はじめに

現在、北埼玉の田舎に在住のエンジニアの卵、エルステリアです。
町工場で機械オペレーターとして従事しているさなか、エクセルのVBAとの出会いからエンジニアを目指して成長していく過程で、作成した制作物をご紹介いたします。

KNC-20G プログラムリスト

製造業時代に、機械(KNC-20Gという名前の機械)で作った製品の記録と、その時使用した機械用のプログラムコードの名前を紐づけて、ログ形式で追加していくことで、再度同じ製品を製造する際のプログラムコードの参照効率性上昇が行えたエクセルVBA。
さらに、新しい記録を追加するときはユーザフォーム経由で行うことで、入力と同時に関連書類の印刷と入力ミス防止に貢献することができました。

- 作成日: 2018 年

- リンク: KNC-20G プログラムリスト.xlsm

NCセット履歴

製造業時代に、10台ある機械のうち、誰が・どの機械で・どの製品を・どれだけ作ったのかを一括管理出来るようにしたエクセルVBA。
これにより参照効率上昇、入力ミスの防止効果が得られました。

- 作成日: 2019 年

- リンク: NCセット履歴.xlsm

Jw_cad図面計測(hou to)

製造業時代に、動画形式のマニュアルを一つのHTMLファイルにまとめたことで閲覧性向上、項目の探索性能上昇に貢献しました。

- 作成日: 2020 年

- リンク: Jw_cad.html

- リンク: GitHub

Attack_on_Mika

職業訓練校時代に、Pythonでフリータイピングソフトの美佳タイプを自動化したもの。
アプリの起動→モード設定→該当文字全ての高速入力　を自動化しました。
ミスタイプはあるものの、コンピュータでの高速入力なので高いスコアを出すことができました。

- 作成日: 2023 年 3 月

- リンク: GitHub

Attack_on_Mika_ver2

Attack_on_Mikaの改良版。
アプリの起動→モード設定→表示文字の読み取り→読み取った文字の入力　を自動化しました。
ミスタイプ無しなので、とても高いスコアを出すことができました。

- 作成日: 2023 年 3 月

- リンク: GitHub

FE単語意味頑張る君

基本情報技術者試験の勉強の過程で出来上がったデスクトップアプリ型エクセルVBA。
ある参考書の索引の、用語・ページ番号・用語の意味を全て書き出したエクセルファイルをもとに、VBAで問題を生成できるようにしました。
問題に対する選択肢も常にランダムなので、選択肢の記号で覚えてしまうことの阻止ができたのと、チャプターごとに解答回数と正答率を算出することで、自身の理解度を過去問よりも正確に測定できました。

- 作成日: 2023 年 4 月

- リンク: FE単語意味頑張る君.xlsm

FE単語意味頑張るマン

FE単語意味頑張る君の、VBAをPythonで書き換えたデスクトップアプリ。
データプール管理はエクセルのままだが、初めてデスクトップアプリを作成できました。EXEファイル化したPythonの起動の遅さを実感し、Pythonでのデスクトップアプリ制作は向いていないことを認識することができました。

- 作成日: 2023 年 5 月

- リンク: GitHub
